PostgreSQL Tutorial

   PostgreSQL Tutorial PostgreSQL Tutorial

   Welcome to the PostgreSQLTutorial.com website! This PostgreSQL tutorial
   helps you understand PostgreSQL quickly. You’ll master PostgreSQL very
   fast through many practical examples and apply the knowledge in
   developing applications using PostgreSQL.

   If you are…
     * Looking for learning PostgreSQL fast.
     * Developing applications using PostgreSQL as the back-end database
       management system.
     * Migrating from other database management systems such as MySQL,
       Oracle, and Microsoft SQL Server to PostgreSQL.

   You’ll find all you need to know to get started with PostgreSQL quickly
   and effectively here on this website.

   PostgreSQL tutorial demonstrates many unique features of PostgreSQL
   that make it the most advanced open-source database management system.

[https://www.postgresqltutorial.com/postgresql-getting-started/]Getting
Started with PostgreSQL

   This section helps you get started with PostgreSQL by showing you how
   to install PostgreSQL on Windows, Linux, and macOS. You also learn how
   to connect to PostgreSQL using the psql tool as well as how to load a
   sample database into the PostgreSQL for practicing.

Basic PostgreSQL Tutorial

   First, you’ll learn how to query data from a single table using basic
   data querying techniques, including selecting data, sorting result
   sets, and filtering rows. Then, you’ll learn about advanced queries
   such as joining multiple tables, using set operations, and constructing
   the subquery. Finally, you will learn how to manage database tables,
   such as creating a new table or modifying an existing table’s
   structure.

Section 1. Querying Data

     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       select/]Select – show you how to query data from a single table.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       column-alias/]Column aliases – learn how to assign temporary names
       to columns or expressions in a query.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       order-by/]Order By – guide you on how to sort the result set
       returned from a query.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       select-distinct/]Select Distinct  – provide you with a clause that
       removes duplicate rows in the result set.

Section 2. Filtering Data

     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       where/]Where – filter rows based on a specified condition.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       limit/]Limit – get a subset of rows generated by a query.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       fetch/]Fetch– limit the number of rows returned by a query.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       in/]In – select data that matches any value in a list of values.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       between/]Between – select data that is a range of values.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       like/]Like – filter data based on pattern matching.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       is-null/]Is Null – check if a value is null or not.

Section 3. Joining Multiple Tables

     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       joins/]Joins – show you a brief overview of joins in PostgreSQL.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       alias/]Table aliases – describes how to use table aliases in the
       query.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       inner-join/]Inner Join – select rows from one table that has the
       corresponding rows in other tables.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       left-join/]Left Join – select rows from one table that may or may
       not have the corresponding rows in other tables.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       self-join/]Self-join – join a table to itself by comparing a table
       to itself.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       full-outer-join/]Full Outer Join – use the full join to find a row
       in a table that does not have a matching row in another table.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       cross-join/]Cross Join – produce a Cartesian product of the rows in
       two or more tables.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       natural-join/]Natural Join – join two or more tables using implicit
       join conditions based on the common column names in the joined
       tables.

Section 4. Grouping Data

     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       group-by/]Group By – divide rows into groups and applies an
       aggregate function on each.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       having/]Having – apply conditions to groups.

Section 5. Set Operations

     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       union/]Union – combine result sets of multiple queries into a
       single result set.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       intersect/]Intersect – combine the result sets of two or more
       queries and returns a single result set that has the rows appear in
       both result sets.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       except/]Except – return the rows in the first query that does not
       appear in the output of the second query.

Section 6. Grouping sets, Cube, and Rollup

     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       grouping-sets/]Grouping Sets  – generate multiple grouping sets in
       reporting.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       cube/]Cube – define multiple grouping sets that include all
       possible combinations of dimensions.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       rollup/]Rollup – generate reports that contain totals and
       subtotals.

Section 7. Subquery

     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       subquery/]Subquery – write a query nested inside another query.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       any/]ANY  – retrieve data by comparing a value with a set of values
       returned by a subquery.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       all/]ALL – query data by comparing a value with a list of values
       returned by a subquery.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       exists/]EXISTS  – check for the existence of rows returned by a
       subquery.

Section 8. Common Table Expressions

     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       cte/]PostgreSQL CTE – introduce you to PostgreSQL common table
       expressions or CTEs.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       recursive-query/]Recursive query using CTEs – discuss the recursive
       query and learn how to apply it in various contexts.

Section 9. Modifying Data

   In this section, you will learn how to insert data into a table with
   the INSERT statement, modify existing data with the UPDATE statement,
   and remove data with the DELETE statement. Besides, you learn how to
   use the upsert statement to merge data.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       insert/]Insert – guide you on how to insert a single row into a
       table.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       insert-multiple-rows/]Insert multiple rows – show you how to insert
       multiple rows into a table.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       update/]Update – update existing data in a table.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       update-join/]Update join – update values in a table based on values
       in another table.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       delete/]Delete – delete data in a table.
     * [https://www.postgresqltutorial.com/postgresql-tutorial/postgresql-
       upsert/]Upsert – insert or update data if the new row already
       exists in the table.
